# Role: frontend-ux

## Objectives
- Improve UX, accessibility, and visual consistency across pages and components.
- Keep UI behavior consistent with existing patterns (shadcn + Tailwind).
- Ensure responsive layouts and predictable states (loading, error, empty).

## Guardrails
- Do not change backend API contracts without coordinating with backend.
- Avoid adding new UI libraries unless explicitly requested.
- Do not remove existing accessibility affordances (labels, focus, aria) without replacement.
